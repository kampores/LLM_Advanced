인공지능과 머신러닝 기초

인공지능(Artificial Intelligence, AI)은 인간의 지능을 모방하여 학습, 추론, 판단 등의 작업을 수행할 수 있는 컴퓨터 시스템을 말합니다. 인공지능은 크게 약인공지능(Narrow AI)과 강인공지능(General AI)으로 구분됩니다. 약인공지능은 특정 작업에 특화된 시스템으로, 이미지 분류, 음성 인식, 자연어 처리 등 정해진 영역에서만 동작합니다. 현재 우리가 사용하는 대부분의 AI 시스템은 약인공지능에 해당합니다. 강인공지능은 인간처럼 다양한 작업을 범용적으로 수행할 수 있는 시스템을 의미하며, 아직 실현되지 않은 목표입니다.

머신러닝(Machine Learning)은 인공지능의 하위 분야로, 명시적인 프로그래밍 없이 데이터로부터 패턴을 학습하는 알고리즘을 연구합니다. 지도학습(Supervised Learning)은 입력-출력 쌍이 주어진 데이터에서 매핑 함수를 학습하며, 분류와 회귀 문제에 사용됩니다. 비지도학습(Unsupervised Learning)은 레이블 없는 데이터에서 구조와 패턴을 발견하며, 클러스터링과 차원 축소가 대표적입니다. 강화학습(Reinforcement Learning)은 환경과 상호작용하며 보상을 최대화하는 정책을 학습합니다. 에이전트가 상태를 관찰하고 행동을 선택하면 환경이 보상과 다음 상태를 반환하는 방식으로 진행됩니다.

딥러닝(Deep Learning)은 머신러닝의 하위 분야로, 인공 신경망(Neural Network)을 여러 층으로 쌓아 복잡한 패턴을 학습합니다. 합성곱 신경망(CNN)은 이미지 처리에 특화되어 있으며, 컨볼루션 필터를 통해 공간적 특징을 추출합니다. 순환 신경망(RNN)은 시퀀스 데이터를 처리하지만, 장기 의존성 문제로 인해 LSTM이나 GRU 같은 변형이 등장했습니다. 트랜스포머(Transformer)는 어텐션 메커니즘을 기반으로 시퀀스 데이터를 병렬 처리할 수 있어 현재 자연어 처리의 표준 아키텍처로 자리 잡았습니다. 딥러닝은 데이터와 계산 자원이 충분할 때 전통적인 머신러닝 방법보다 우수한 성능을 보이며, 특히 비정형 데이터(텍스트, 이미지, 음성)에서 강점을 가집니다.

자연어 처리와 대규모 언어 모델

자연어 처리(Natural Language Processing, NLP)는 컴퓨터가 인간의 언어를 이해하고 생성할 수 있게 하는 기술입니다. 토큰화, 품사 태깅, 개체명 인식, 감정 분석, 기계 번역, 텍스트 요약, 질의응답 등이 주요 태스크입니다. 초기에는 규칙 기반 접근이 주류였으나, 통계적 방법을 거쳐 현재는 딥러닝 기반 방법이 대부분의 태스크에서 최고 성능을 보이고 있습니다. 특히 사전 학습과 파인튜닝 패러다임이 등장하면서 NLP의 성능은 비약적으로 향상되었습니다.

대규모 언어 모델(Large Language Model, LLM)은 수십억 개 이상의 파라미터를 가진 언어 모델로, 방대한 텍스트 데이터로 사전 학습됩니다. GPT 시리즈는 OpenAI에서 개발한 디코더 기반 모델로, GPT-3(175B), GPT-4에 이르기까지 성능을 지속적으로 향상시켜 왔습니다. Meta의 LLaMA 시리즈는 오픈소스 모델의 혁신을 이끌었으며, 7B부터 70B까지 다양한 크기로 제공됩니다. 알리바바의 Qwen 시리즈는 0.5B부터 72B까지 폭넓은 크기를 지원하며, 특히 다국어 성능이 우수합니다. Google의 Gemma는 경량 오픈소스 모델로 2B와 7B 크기로 제공됩니다. 이러한 LLM들은 텍스트 생성, 질의응답, 코드 작성, 논리적 추론 등 다양한 작업을 수행할 수 있으며, 스케일링 법칙(Scaling Law)에 따라 모델 크기, 데이터 양, 계산량을 늘릴수록 성능이 향상되는 경향이 있습니다.

소규모 언어 모델(sLLM, Small Language Model)은 1B에서 7B 사이의 파라미터를 가진 모델을 지칭합니다. 온디바이스 배포, 빠른 추론 속도, 낮은 비용 등의 장점으로 실무 환경에서 주목받고 있습니다. Qwen2.5-1.5B, Phi-3-mini, Gemma-2B 등이 대표적이며, 도메인 특화 파인튜닝을 통해 특정 분야에서 대형 모델에 근접한 성능을 달성할 수 있습니다.

트랜스포머 아키텍처 상세

트랜스포머(Transformer)는 2017년 Google이 "Attention is All You Need" 논문에서 제안한 아키텍처입니다. 셀프 어텐션(Self-Attention) 메커니즘을 핵심으로 하며, 입력 시퀀스의 모든 위치를 동시에 처리할 수 있어 병렬화가 가능합니다. 원래 인코더-디코더 구조로 제안되었으나, 현대 LLM 대부분은 디코더만 사용하는 구조를 채택하고 있습니다.

어텐션 메커니즘은 Query, Key, Value 세 가지 벡터를 사용합니다. Query와 Key의 내적(dot product)으로 어텐션 스코어를 계산하고, 차원의 제곱근으로 나누어 스케일링한 뒤 Softmax를 적용하여 가중치를 구합니다. 이 가중치를 Value에 곱하여 최종 출력을 생성합니다. 수식으로 표현하면 Attention(Q,K,V) = softmax(QK^T / sqrt(d_k))V 입니다.

멀티헤드 어텐션(Multi-Head Attention)은 여러 개의 어텐션 헤드를 병렬로 수행하여 서로 다른 관점에서 입력을 분석합니다. 각 헤드는 독립적인 Q, K, V 투영을 학습하며, 이를 통해 모델은 다양한 종류의 관계를 동시에 학습할 수 있습니다. 예를 들어, 한 헤드는 구문적 관계를, 다른 헤드는 의미적 관계를 포착할 수 있습니다.

포지셔널 인코딩(Positional Encoding)은 트랜스포머가 입력 토큰의 순서 정보를 파악할 수 있게 해주는 기법입니다. 원래 논문에서는 사인과 코사인 함수를 사용한 절대 위치 인코딩을 제안했습니다. 그러나 현대 LLM에서는 RoPE(Rotary Position Embedding)가 널리 사용됩니다. RoPE는 회전 행렬을 사용하여 토큰의 위치 정보를 임베딩 벡터에 통합하며, 상대적 위치 관계를 자연스럽게 인코딩합니다. 두 토큰 간의 어텐션 스코어가 오직 상대적 거리에만 의존하도록 설계되어 있어, 학습 시보다 더 긴 시퀀스에도 외삽(extrapolation)이 가능합니다. Qwen2.5, LLaMA 3 등 최신 모델들은 모두 RoPE를 채택하고 있습니다.

레이어 노멀라이제이션(Layer Normalization)은 학습 안정성을 확보하기 위한 정규화 기법입니다. 원래 트랜스포머에서는 Post-Norm 구조를 사용했으나, 현대 LLM은 Pre-Norm 구조를 채택하여 잔차 연결(residual connection) 이전에 정규화를 수행합니다. 특히 RMSNorm(Root Mean Square Normalization)이 널리 사용되는데, 기존 LayerNorm에서 평균을 빼는 중심화(centering) 연산을 제거하고 RMS(제곱 평균의 제곱근) 값으로만 정규화합니다. 이로 인해 계산량이 줄어들면서도 성능은 동등하게 유지되어, LLaMA, Qwen 등 대부분의 최신 모델이 RMSNorm을 사용합니다.

피드포워드 네트워크(Feed-Forward Network, FFN)는 각 트랜스포머 블록에서 어텐션 이후에 위치하며, 비선형 변환을 수행합니다. 원래 트랜스포머에서는 두 개의 선형 레이어와 ReLU 활성화 함수를 사용했습니다. 현대 LLM에서는 SwiGLU(Swish-Gated Linear Unit) 활성화 함수가 표준입니다. SwiGLU는 게이트 메커니즘과 Swish 활성화를 결합한 것으로, 입력을 두 갈래로 나누어 하나에는 Swish 함수를 적용하고 다른 하나와 원소별 곱을 수행합니다. SwiGLU를 사용하면 모델의 표현력이 향상되어 같은 파라미터 수에서 더 나은 성능을 달성합니다. 다만 게이트로 인해 파라미터 수가 약간 증가하므로, FFN의 중간 차원을 조정하여 전체 파라미터 수를 맞춥니다.

사전학습 과정

대규모 언어 모델의 사전학습(Pretraining)은 Next Token Prediction을 핵심 목표로 합니다. 주어진 문맥에서 다음에 올 토큰을 예측하도록 학습하며, 이 과정에서 모델은 언어의 문법, 사실 관계, 논리적 추론 능력 등을 자연스럽게 습득합니다. 손실 함수로는 Cross-Entropy Loss를 사용하며, 모델이 출력하는 토큰 확률 분포와 실제 다음 토큰 간의 차이를 최소화합니다.

사전학습 데이터는 웹 크롤링, 서적, 학술 논문, 코드 저장소 등 다양한 출처에서 수집됩니다. Common Crawl, The Pile, RedPajama, FineWeb 등의 대규모 데이터셋이 사용됩니다. 데이터 품질이 모델 성능에 결정적인 영향을 미치므로, 중복 제거(deduplication), 품질 필터링, 유해 콘텐츠 제거, 언어별 균형 조정 등의 전처리가 필수적입니다. 최근 연구에서는 데이터의 양보다 품질이 더 중요하다는 결과가 반복적으로 보고되고 있으며, Phi-3와 같은 모델은 고품질 데이터 큐레이션을 통해 작은 모델로도 우수한 성능을 달성했습니다.

학습 인프라 측면에서는 수백에서 수천 개의 GPU 또는 TPU를 사용한 분산 학습이 필요합니다. 데이터 병렬화(Data Parallelism)는 동일한 모델을 여러 GPU에 복제하고 데이터를 분할하여 처리합니다. 모델 병렬화(Model Parallelism)는 모델 자체를 여러 GPU에 분할합니다. 텐서 병렬화(Tensor Parallelism)는 개별 연산을 여러 GPU에 분산하고, 파이프라인 병렬화(Pipeline Parallelism)는 모델의 레이어를 여러 GPU에 순차적으로 배치합니다. ZeRO(Zero Redundancy Optimizer)는 옵티마이저 상태, 그래디언트, 파라미터를 GPU 간에 분산하여 메모리 효율을 높입니다. DeepSpeed와 Megatron-LM 같은 프레임워크가 이러한 분산 학습을 지원합니다.

학습률 스케줄링(Learning Rate Scheduling)은 학습 과정에서 학습률을 동적으로 조절하는 전략입니다. 일반적으로 워밍업(warmup) 단계에서 학습률을 점진적으로 증가시킨 후, 코사인 스케줄링(cosine scheduling)을 적용하여 서서히 감소시킵니다. 워밍업은 학습 초기의 불안정성을 방지하며, 코사인 감쇠는 학습 후반에 세밀한 조정이 가능하게 합니다. 최대 학습률, 워밍업 비율, 최소 학습률 등이 주요 하이퍼파라미터입니다.

토큰화와 임베딩

토큰화(Tokenization)는 텍스트를 모델이 처리할 수 있는 작은 단위(토큰)로 분할하는 과정입니다. 토큰화의 품질은 모델 성능에 직접적인 영향을 미칩니다. 좋은 토크나이저는 자주 사용되는 단어는 하나의 토큰으로, 드문 단어는 여러 서브워드로 분할하여 어휘 크기와 시퀀스 길이 사이의 균형을 맞춥니다.

BPE(Byte Pair Encoding)는 가장 널리 사용되는 토큰화 알고리즘입니다. 먼저 텍스트를 개별 문자(또는 바이트)로 분할한 후, 가장 빈번하게 등장하는 문자 쌍을 반복적으로 병합하여 어휘(vocabulary)를 구축합니다. GPT 시리즈에서 사용하는 tiktoken은 BPE의 효율적인 구현체입니다. WordPiece는 BERT에서 사용되며, 우도(likelihood)를 최대화하는 방향으로 병합을 수행합니다. SentencePiece는 언어에 독립적인 토크나이저로, 공백을 특수 토큰으로 처리하여 한국어, 일본어 등 공백이 불규칙한 언어에도 적용할 수 있습니다.

한국어 토큰화에서 특히 중요한 점은 토큰 효율성입니다. 영어 중심으로 학습된 토크나이저는 한국어 텍스트를 처리할 때 같은 의미의 문장도 영어보다 2~3배 많은 토큰으로 분할하는 경향이 있습니다. 이는 추론 비용 증가와 컨텍스트 윈도우 낭비로 이어집니다. Qwen이나 LLaMA 3처럼 다국어 데이터를 충분히 포함하여 학습한 토크나이저는 한국어 효율이 상대적으로 높습니다.

임베딩(Embedding)은 토큰을 고차원 벡터 공간에 매핑하는 과정입니다. 의미적으로 유사한 토큰은 벡터 공간에서 가까운 위치에 놓이게 됩니다. Word2Vec은 CBOW와 Skip-gram 방식으로 단어 임베딩을 학습하며, GloVe는 공출현 행렬의 통계 정보를 활용합니다. FastText는 서브워드 정보를 활용하여 미등록 단어도 처리할 수 있습니다. 현대 LLM은 트랜스포머 내부에서 문맥을 고려한 임베딩을 학습하며, 같은 단어라도 주변 문맥에 따라 다른 임베딩 벡터를 생성합니다. 문장 임베딩 모델인 SentenceTransformer는 전체 문장의 의미를 하나의 벡터로 표현하며, 시맨틱 서치와 RAG에서 핵심적인 역할을 합니다.

파인튜닝과 효율적 학습

파인튜닝(Fine-tuning)은 사전 학습된 모델을 특정 태스크나 도메인에 맞게 추가 학습하는 과정입니다. 전체 파라미터를 업데이트하는 Full Fine-tuning(FFT)은 모든 가중치를 수정하므로 최대한의 유연성을 제공하지만, 모델 전체를 메모리에 올리고 그래디언트까지 저장해야 하므로 막대한 GPU 메모리가 필요합니다. 7B 모델의 FFT에는 약 60GB 이상의 VRAM이 필요할 수 있습니다.

Parameter-Efficient Fine-tuning(PEFT)은 모델의 소수 파라미터만 학습하여 효율적으로 파인튜닝하는 접근법입니다. LoRA, Prefix Tuning, Prompt Tuning, Adapter 등 다양한 방법이 있으며, 그중 LoRA가 가장 널리 사용됩니다. PEFT의 핵심 가정은 파인튜닝 과정에서의 가중치 변화가 낮은 랭크(low-rank)로 근사할 수 있다는 것입니다.

Continuous Pretraining(지속 사전학습)은 사전 학습된 모델에 도메인 특화 텍스트를 추가로 학습시키는 방법입니다. 의료, 법률, 금융 등 특정 도메인의 지식을 주입하는 데 사용됩니다. 일반 텍스트를 Next Token Prediction 방식으로 학습하며, 학습률을 사전학습 때보다 낮게 설정하여 기존 지식의 망각(catastrophic forgetting)을 최소화합니다. 도메인 텍스트와 일반 텍스트를 적절한 비율로 혼합하여 학습하는 것이 효과적입니다.

Instruction Tuning(지시 튜닝)은 모델이 사용자의 지시를 잘 따르도록 학습하는 방법입니다. instruction-input-output 형식의 데이터를 사용하며, Alpaca 형식은 단일 턴 지시-응답 쌍을, ShareGPT 형식은 멀티턴 대화를 표현합니다. ChatML은 role과 content를 명시적으로 구분하는 형식으로, 시스템 프롬프트, 사용자 메시지, 어시스턴트 응답을 구조화합니다. 지시 튜닝을 거친 모델은 사전 학습 모델보다 사용자 의도를 잘 파악하고 유용한 응답을 생성합니다.

SFT(Supervised Fine-Tuning)에서 중요한 하이퍼파라미터로는 학습률, 배치 크기, 에폭 수, 최대 시퀀스 길이 등이 있습니다. RTX 4060(8GB VRAM) 같은 소비자급 GPU에서는 batch_size=1, gradient_accumulation_steps=8, gradient_checkpointing=True 설정이 권장됩니다. 그래디언트 체크포인팅은 순전파 시 중간 활성화를 저장하지 않고 역전파 시 재계산하여 메모리를 절약하는 기법으로, 학습 속도가 약간 느려지는 대신 메모리 사용량을 크게 줄일 수 있습니다.

LoRA와 QLoRA 상세 원리

LoRA(Low-Rank Adaptation)는 가장 인기 있는 PEFT 방법으로, 저랭크 분해(low-rank decomposition)를 핵심 원리로 합니다. 원본 모델의 가중치 행렬 W에 대한 업데이트를 직접 학습하는 대신, 업데이트를 두 개의 작은 행렬 A와 B의 곱(BA)으로 분해합니다. 원본 가중치 W(d x d)를 동결하고, B(d x r)와 A(r x d) 행렬만 학습합니다. 여기서 r은 랭크로, d보다 훨씬 작은 값(보통 4~64)을 사용합니다. 추론 시에는 W + BA를 미리 계산하여 원본 모델과 동일한 속도로 추론할 수 있습니다.

r(rank) 값은 LoRA의 표현력을 결정하는 핵심 하이퍼파라미터입니다. r이 클수록 더 복잡한 변환을 학습할 수 있지만, 학습 파라미터 수와 메모리 사용량이 증가합니다. 실무에서는 r=8~32가 가장 많이 사용되며, r=16이 성능과 효율의 균형점으로 권장됩니다. 복잡한 태스크나 큰 모델에서는 r=32~64를 사용하기도 합니다.

alpha 값은 LoRA 업데이트의 스케일링 팩터입니다. 실제 업데이트는 (alpha/r) * BA로 스케일링되므로, alpha가 클수록 LoRA의 영향이 커집니다. 관례적으로 alpha는 r의 2배(예: r=16이면 alpha=32)로 설정합니다. 이 비율을 유지하면 r을 변경해도 실효 학습률이 일관되게 유지됩니다.

target modules는 LoRA를 적용할 레이어를 지정합니다. 트랜스포머의 어텐션 레이어(q_proj, k_proj, v_proj, o_proj)에 적용하는 것이 기본이며, 피드포워드 레이어(gate_proj, up_proj, down_proj)까지 확장하면 성능이 향상되는 경우가 많습니다. 모든 선형 레이어에 적용하면 학습 파라미터가 증가하지만, FFT에 더 가까운 성능을 달성할 수 있습니다. lora_dropout은 과적합 방지를 위한 드롭아웃 비율로, 0.05~0.1 정도가 일반적입니다.

QLoRA(Quantized LoRA)는 LoRA에 양자화를 결합한 방법으로, 메모리 효율을 극대화합니다. 기본 모델을 NF4(4-bit NormalFloat) 양자화로 로드하여 메모리를 약 75% 절약하면서, LoRA 어댑터는 FP16/BF16 정밀도로 학습합니다. NF4는 정규 분포를 따르는 가중치에 최적화된 데이터 타입으로, 일반적인 INT4보다 정보 손실이 적습니다. 이중 양자화(Double Quantization)는 양자화 상수 자체를 다시 양자화하여 추가 메모리를 절약합니다. QLoRA를 사용하면 8GB VRAM의 GPU에서도 7B 모델의 파인튜닝이 가능하며, 연구 결과에 따르면 Full Fine-tuning과 비교해도 성능 손실이 미미합니다.

양자화 상세

양자화(Quantization)는 모델의 가중치를 더 낮은 정밀도로 변환하여 메모리 사용량과 추론 지연시간을 줄이는 핵심 기술입니다. 다양한 수치 정밀도가 사용되며, 각각 장단점이 있습니다.

FP32(32비트 부동소수점)는 학습의 기본 정밀도로, 부호 1비트, 지수 8비트, 가수 23비트로 구성됩니다. 높은 정밀도를 제공하지만 메모리와 계산 비용이 큽니다. 7B 모델은 FP32로 약 28GB의 메모리를 차지합니다. FP16(16비트 부동소수점)은 부호 1비트, 지수 5비트, 가수 10비트로, FP32 대비 메모리를 절반으로 줄이면서 대부분의 딥러닝 작업에서 충분한 정밀도를 제공합니다. 다만 표현 범위가 좁아 학습 시 오버플로우/언더플로우가 발생할 수 있습니다.

BF16(Brain Floating Point 16)은 Google에서 제안한 형식으로, 부호 1비트, 지수 8비트, 가수 7비트로 구성됩니다. FP32와 동일한 지수 범위를 가져 오버플로우 문제가 적으며, 학습 안정성이 FP16보다 우수합니다. NVIDIA A100, H100 등의 최신 GPU와 AMD MI300에서 하드웨어 가속을 지원합니다. 다만 NVIDIA TITAN RTX나 RTX 4060에서는 BF16 하드웨어 가속이 제한적이므로 FP16을 사용하는 것이 효율적입니다.

INT8(8비트 정수)은 모델 크기를 FP16 대비 절반으로 줄이며, LLM.int8() 같은 기법을 통해 추론 품질을 유지하면서 메모리를 절약합니다. 이 기법은 이상치(outlier) 특성을 가진 채널만 FP16으로 처리하고 나머지를 INT8로 양자화하는 혼합 정밀도 방식을 사용합니다.

INT4(4비트 정수) 양자화는 모델 크기를 FP16 대비 약 75%까지 줄입니다. NF4(4-bit NormalFloat)는 사전 학습된 모델의 가중치가 대략 정규 분포를 따른다는 관찰에 기반하여, 정규 분포의 분위수를 양자화 레벨로 사용합니다. 이로 인해 동일한 4비트라도 일반 INT4보다 정보 보존이 우수합니다.

GPTQ(Generative Pre-trained Transformer Quantization)는 학습 후 양자화(Post-Training Quantization) 기법으로, 소량의 캘리브레이션 데이터를 사용하여 양자화 오류를 최소화합니다. 각 가중치 행렬을 순서대로 처리하며, 양자화로 인한 출력 오류를 미양자화 가중치로 보상하는 방식을 사용합니다. GPU 추론에 최적화되어 있으며, AutoGPTQ 라이브러리로 쉽게 적용할 수 있습니다.

AWQ(Activation-aware Weight Quantization)는 활성화의 중요도를 고려하여 양자화하는 기법입니다. 활성화 값이 큰 채널의 가중치는 양자화에 더 민감하므로, 해당 채널에 스케일링 팩터를 적용하여 양자화 오류를 줄입니다. GPTQ보다 캘리브레이션 시간이 짧고, 일부 벤치마크에서 더 나은 성능을 보입니다.

GGUF(GPT-Generated Unified Format)는 llama.cpp에서 사용하는 양자화 포맷으로, CPU와 GPU 혼합 추론을 지원합니다. Q4_K_M, Q5_K_M 등 다양한 양자화 수준을 제공하며, Ollama와 같은 로컬 추론 도구에서 주로 사용됩니다. CPU에서도 합리적인 속도로 추론이 가능하여 엣지 디바이스 배포에 적합합니다.

강화학습과 선호도 최적화

RLHF(Reinforcement Learning from Human Feedback)는 인간의 선호도를 반영하여 모델을 개선하는 방법입니다. 먼저 SFT를 통해 기본적인 지시 수행 능력을 갖춘 모델을 만든 후, 인간 평가자가 모델 출력을 비교하여 선호도 데이터를 수집합니다. 이 데이터로 보상 모델(Reward Model)을 학습한 후, PPO(Proximal Policy Optimization) 알고리즘으로 보상을 최대화하는 방향으로 정책을 최적화합니다. PPO는 정책 업데이트의 크기를 제한하여 학습 안정성을 확보하지만, 보상 모델 학습, 참조 모델 유지, PPO 자체의 복잡성 등으로 인해 구현과 학습이 까다롭습니다.

DPO(Direct Preference Optimization)는 보상 모델 없이 직접 선호도를 학습하는 방법입니다. 직관적으로 설명하면, DPO는 모델에게 좋은 답변(chosen)의 확률은 높이고 나쁜 답변(rejected)의 확률은 낮추도록 학습시킵니다. 수학적으로는 보상 함수를 정책 자체로 재매개변수화하여, RLHF의 최적화 목표를 단순한 분류 손실로 변환합니다. 참조 모델과의 비교를 통해 모델이 극단적으로 변하는 것을 방지합니다. PPO보다 구현이 간단하고 학습이 안정적이며, 보상 모델을 별도로 학습할 필요가 없어 메모리와 계산 비용이 절약됩니다. 다만 오프라인 데이터에 의존하므로, 선호도 데이터의 품질이 결과에 직접적인 영향을 미칩니다.

GRPO(Group Relative Policy Optimization)는 DeepSeek에서 제안한 방법입니다. 직관적으로, GRPO는 하나의 질문에 대해 여러 답변을 동시에 생성한 후, 그 답변들 사이에서 상대적으로 좋은 것은 강화하고 나쁜 것은 약화시킵니다. 외부 보상 모델이나 Critic 모델이 필요 없이, 그룹 내 답변들의 상대적 보상만으로 학습합니다. 이는 규칙 기반 보상(정답 여부, 형식 준수 등)을 활용할 수 있어 수학 문제나 코딩 태스크에서 특히 효과적입니다. DeepSeek-R1은 GRPO를 사용하여 추론 능력을 강화한 대표적인 사례입니다. 메모리 효율이 높아 소규모 GPU 환경에서도 적용 가능하다는 장점이 있습니다.

Rejection Sampling은 모델이 여러 응답을 생성한 후, 보상 모델이나 규칙 기반 평가로 최적의 응답만 선별하여 SFT 데이터로 활용하는 방법입니다. 구현이 단순하면서도 효과적이며, DPO나 PPO를 적용하기 전 단계로 사용되기도 합니다.

데이터 품질과 합성 데이터

파인튜닝에서 데이터의 품질은 양보다 훨씬 중요합니다. LIMA 논문에서 보여주듯이, 잘 큐레이션된 1,000개의 고품질 데이터가 수만 건의 저품질 데이터보다 더 나은 결과를 만들 수 있습니다. 데이터 품질 관리의 핵심 요소로는 정확성(사실 관계의 올바름), 일관성(형식과 스타일의 통일), 다양성(입력 패턴의 분포), 완전성(필수 필드의 누락 여부) 등이 있습니다.

데이터 수집은 기존 데이터셋 활용, 수동 제작, 크라우드소싱, 합성 데이터 생성 등의 방법으로 이루어집니다. 공개 데이터셋으로는 Alpaca(52K), ShareGPT, Open-Orca 등이 있으며, 한국어 데이터셋으로는 KoAlpaca, KULLM 등이 있습니다. 수동으로 데이터를 제작할 때는 도메인 전문가의 참여가 중요하며, 레이블링 가이드라인을 명확히 정의해야 합니다.

Self-Instruct는 LLM을 활용하여 학습 데이터를 자동 생성하는 기법입니다. 소수의 시드(seed) 데이터에서 시작하여, 모델이 새로운 지시와 응답을 생성하고, 품질 필터링을 거쳐 데이터셋을 확장합니다. 예를 들어, 10건의 시드 데이터에서 시작하여 100건, 1000건으로 확장할 수 있습니다. 다양성을 유지하기 위해 카테고리별 균형 조정과 유사도 필터링이 중요합니다.

Distillation(증류)은 대형 모델(teacher)의 지식을 소형 모델(student)에 전달하는 기법입니다. 세 가지 주요 전략이 있습니다. 첫째, 출력 증류는 teacher 모델의 응답을 그대로 student의 학습 데이터로 사용합니다. 둘째, 로짓 증류는 teacher의 출력 확률 분포를 student가 모방하도록 학습합니다. 셋째, 특징 증류는 중간 레이어의 표현을 전달합니다. GPT-4로 생성한 데이터로 7B 모델을 학습하는 것이 출력 증류의 대표적인 예입니다. 비용 관점에서 GPT-4 API로 1만 건의 합성 데이터를 생성하는 비용은 수십 달러 수준으로, 인간 어노테이터를 고용하는 것보다 훨씬 경제적입니다.

데이터 정제(Data Cleaning)는 수집된 데이터의 품질을 높이는 과정입니다. 중복 제거, 길이 필터링, 언어 감지, 형식 검증, 유해 콘텐츠 필터링, 개인정보 마스킹 등의 단계를 거칩니다. 자동화된 파이프라인을 구축하여 일관된 품질 관리를 수행하는 것이 효과적입니다.

프롬프트 엔지니어링

프롬프트 엔지니어링(Prompt Engineering)은 LLM에서 원하는 출력을 얻기 위해 입력 프롬프트를 설계하는 기법입니다. 모델의 파라미터를 변경하지 않고도 출력 품질을 크게 향상시킬 수 있어, 가장 비용 효율적인 성능 개선 방법 중 하나입니다.

제로샷(Zero-shot) 프롬프팅은 별도의 예시 없이 지시만으로 모델에게 태스크를 수행시키는 방법입니다. 예를 들어 "다음 문장의 감정을 긍정/부정으로 분류하세요"와 같이 직접적으로 요청합니다. 충분히 큰 모델은 사전학습 과정에서 습득한 지식만으로도 다양한 태스크를 수행할 수 있습니다.

퓨샷(Few-shot) 프롬프팅은 몇 개의 입출력 예시를 프롬프트에 포함하여 모델이 패턴을 파악하도록 하는 방법입니다. 예시의 수가 많을수록 성능이 향상되는 경향이 있지만, 컨텍스트 윈도우 제한이 있으므로 3~5개의 대표적인 예시를 선택하는 것이 일반적입니다. 예시의 다양성과 순서도 결과에 영향을 미칩니다.

Chain of Thought(CoT) 프롬프팅은 모델에게 단계별로 사고 과정을 서술하도록 유도하는 기법입니다. "단계별로 생각해봅시다(Let's think step by step)"와 같은 간단한 구문을 추가하는 것만으로도 수학 문제나 논리적 추론 태스크에서 성능이 크게 향상됩니다. 더 정교한 방법으로는 사고 과정의 예시를 포함하는 수동 CoT가 있습니다.

시스템 프롬프트(System Prompt)는 모델의 전반적인 행동 방식, 역할, 제약 조건을 설정하는 고정 프롬프트입니다. 대화의 시작에 위치하며, 모든 후속 상호작용에 영향을 미칩니다. 효과적인 시스템 프롬프트는 모델의 역할을 명확히 정의하고, 응답 형식을 지정하며, 제약 조건을 구체적으로 명시합니다. 예를 들어 "당신은 한국어 의료 상담 전문가입니다. 환자의 질문에 정확하고 이해하기 쉬운 한국어로 답변하세요"와 같이 설정합니다.

기타 프롬프트 기법으로는 출력 형식 지정(JSON, 표, 불릿 포인트 등), 역할 부여(전문가, 교사, 비평가 등), 자기 일관성(Self-Consistency, 여러 응답을 생성한 후 다수결로 결정), 구조화된 프롬프트(섹션별로 구분하여 복잡한 지시를 전달) 등이 있습니다.

RAG과 벡터 데이터베이스

RAG(Retrieval-Augmented Generation)는 외부 지식 베이스에서 관련 정보를 검색하여 LLM의 답변을 보강하는 기술입니다. LLM의 환각(hallucination) 문제를 완화하고, 학습 데이터에 포함되지 않은 최신 정보나 도메인 특화 지식을 반영할 수 있습니다. 기본 RAG 파이프라인은 문서 로딩, 청킹, 임베딩, 벡터 저장, 검색, 생성의 단계로 구성됩니다.

청킹(Chunking)은 긴 문서를 검색에 적합한 크기의 단위로 분할하는 과정입니다. 고정 크기 청킹은 일정한 글자 수 또는 토큰 수로 분할하며, 구현이 간단합니다. 재귀적 문자 분할(Recursive Character Splitting)은 문단, 문장, 단어 순서로 계층적으로 분할하여 의미 단위를 최대한 유지합니다. 시맨틱 청킹은 임베딩 유사도를 기준으로 의미적으로 응집된 단위로 분할합니다. 청크 크기(보통 200~1000 토큰)와 오버랩(인접 청크 간 중복, 보통 50~200 토큰)은 검색 성능에 큰 영향을 미치는 핵심 파라미터입니다.

임베딩 모델은 텍스트를 벡터로 변환하는 역할을 합니다. SentenceTransformer 기반의 모델이 널리 사용되며, all-MiniLM-L6-v2는 경량이면서도 우수한 성능을 보입니다. 다국어 모델로는 multilingual-e5-large, BGE-M3 등이 있으며, 한국어에 특화된 모델도 있습니다. OpenAI의 text-embedding-3-small/large와 같은 API 기반 임베딩 모델도 활용됩니다.

벡터 데이터베이스는 텍스트를 임베딩 벡터로 변환하여 저장하고, 코사인 유사도, 유클리드 거리, 내적(dot product) 등으로 유사한 문서를 빠르게 검색합니다. ChromaDB는 로컬 환경에서 가볍게 사용할 수 있는 오픈소스 벡터 데이터베이스입니다. Pinecone은 클라우드 기반 관리형 서비스로 확장성이 뛰어납니다. Weaviate는 하이브리드 검색(벡터+키워드)을 네이티브로 지원합니다. Milvus는 대규모 데이터셋에 최적화된 분산 벡터 데이터베이스입니다. FAISS는 Meta에서 개발한 유사도 검색 라이브러리로, 벡터 데이터베이스의 내부 엔진으로 활용되기도 합니다.

리랭킹(Reranking)은 초기 검색 결과를 재정렬하여 가장 관련성 높은 문서를 상위에 배치하는 기법입니다. CrossEncoder는 질문과 문서를 하나의 입력으로 결합하여 관련성 점수를 직접 계산합니다. 바이 인코더(Bi-Encoder) 기반 검색보다 정확하지만 계산 비용이 높으므로, 보통 상위 10~20개 후보에 대해서만 적용합니다.

Advanced RAG 기법으로는 HyDE(Hypothetical Document Embeddings), Ensemble Retriever, Parent Document Retriever 등이 있습니다. HyDE는 질문을 직접 검색하는 대신, LLM으로 가상의 답변 문서를 생성한 후 이를 임베딩하여 검색합니다. 질문과 문서 사이의 의미적 간극을 줄여 검색 품질을 향상시킵니다. Ensemble Retriever는 BM25(키워드 기반)와 시맨틱 검색을 결합하여 두 방법의 장점을 모두 활용합니다. Parent Document Retriever는 작은 청크로 검색하되, 실제로는 해당 청크를 포함하는 더 큰 문서를 반환하여 충분한 문맥을 제공합니다.

sLLM과 모델 경량화

sLLM(Small Language Model)은 수십억 파라미터 이하의 언어 모델을 지칭하며, 실용적인 배포와 추론 효율에 초점을 맞춥니다. 대형 모델의 성능에는 미치지 못하지만, 낮은 비용, 빠른 추론 속도, 로컬 실행 가능성, 개인정보 보호 등의 장점이 있습니다. 특히 도메인 특화 파인튜닝을 적용하면 해당 분야에서 대형 모델에 근접한 성능을 달성할 수 있어, 기업 환경에서 주목받고 있습니다.

GQA(Grouped Query Attention)는 추론 효율을 높이기 위한 어텐션 메커니즘의 변형입니다. 기존 Multi-Head Attention에서는 모든 헤드가 각각의 Key와 Value를 가지지만, GQA에서는 여러 Query 헤드가 Key와 Value를 공유합니다. 예를 들어 32개의 Query 헤드를 8개의 Key-Value 그룹으로 묶으면, KV 캐시 크기가 4분의 1로 줄어들어 메모리 효율이 크게 향상됩니다. 극단적인 경우인 MQA(Multi-Query Attention)는 모든 헤드가 하나의 Key-Value를 공유합니다. LLaMA 2(70B), Qwen2.5, Mistral 등 대부분의 최신 모델이 GQA를 채택하고 있습니다.

Flash Attention은 어텐션 연산의 메모리 효율과 속도를 동시에 개선하는 알고리즘입니다. 기존 어텐션은 전체 어텐션 행렬을 GPU의 고대역폭 메모리(HBM)에 저장하여 메모리 사용량이 시퀀스 길이의 제곱에 비례합니다. Flash Attention은 타일링(tiling) 기법으로 어텐션을 블록 단위로 계산하여, SRAM(온칩 메모리)을 활용하고 HBM 접근을 최소화합니다. 이를 통해 메모리 사용량을 선형으로 줄이면서 벽시계 시간(wall-clock time)도 단축합니다. Flash Attention 2는 병렬성을 더욱 개선하였으며, 대부분의 최신 LLM 프레임워크에서 기본으로 지원됩니다.

온디바이스 AI(On-device AI)는 클라우드가 아닌 스마트폰, 랩톱, 엣지 디바이스에서 직접 AI 모델을 실행하는 패러다임입니다. 네트워크 지연이 없고 개인정보가 디바이스를 떠나지 않으며 오프라인에서도 동작합니다. 이를 위해서는 모델 크기와 추론 속도의 극단적인 최적화가 필요합니다. 양자화(INT4/INT8), 지식 증류, 구조적 가지치기(pruning) 등의 기법이 결합되어 사용됩니다. Apple의 CoreML, Google의 MediaPipe, Qualcomm의 AI Engine 등이 온디바이스 추론을 지원하는 프레임워크입니다. llama.cpp는 CPU에서 효율적으로 LLM을 실행할 수 있게 해주는 오픈소스 도구로, GGUF 양자화 모델과 함께 온디바이스 시나리오에서 널리 사용됩니다.

모델 서빙과 최적화

모델 서빙은 학습된 모델을 실제 서비스 환경에서 효율적으로 운영하는 과정입니다. 추론 최적화, 동시 요청 처리, 확장성, 안정성 등이 핵심 과제입니다.

vLLM은 PagedAttention을 사용하여 높은 처리량의 LLM 서빙을 제공하는 프레임워크입니다. PagedAttention은 운영체제의 가상 메모리 페이징에서 영감을 받아, KV 캐시를 고정 크기의 블록으로 관리합니다. 이를 통해 KV 캐시의 메모리 낭비를 줄이고, 동시에 많은 요청을 처리할 수 있습니다. 연속 배칭(continuous batching)을 지원하여, 완료된 요청의 자리에 즉시 새 요청을 투입함으로써 GPU 활용률을 극대화합니다. OpenAI 호환 API를 제공하므로 기존 OpenAI 클라이언트 코드를 거의 수정 없이 재사용할 수 있습니다.

Ollama는 로컬 환경에서 LLM을 쉽게 실행할 수 있게 해주는 도구입니다. 명령줄에서 간단하게 모델을 다운로드하고 실행할 수 있으며, GGUF 포맷의 양자화 모델을 지원합니다. REST API를 통해 다른 애플리케이션과 연동할 수 있고, 모델 파일(Modelfile)을 통해 사용자 정의 모델을 생성할 수 있습니다. 개발과 프로토타이핑에 특히 유용하며, 개인정보 보호가 필요한 환경에서 로컬 추론을 가능하게 합니다.

KV 캐시(Key-Value Cache)는 자기회귀(autoregressive) 생성에서 이전 토큰의 Key와 Value를 캐싱하여 중복 계산을 방지하는 기법입니다. 시퀀스가 길어질수록 KV 캐시의 메모리 사용량이 증가하므로, 효율적인 관리가 중요합니다. GQA를 통한 KV 헤드 수 감소, 양자화된 KV 캐시, 슬라이딩 윈도우 어텐션 등이 KV 캐시 최적화 방법입니다.

스펙큘러티브 디코딩(Speculative Decoding)은 작은 드래프트(draft) 모델이 빠르게 후보 토큰을 생성하고, 큰 모델이 이를 병렬로 검증하는 방식으로 추론 속도를 높이는 기법입니다. 드래프트 모델의 예측이 맞으면 여러 토큰을 한 번에 수용하여 처리량이 향상됩니다.

추론 서버 구축 시에는 FastAPI나 Flask로 API 엔드포인트를 만들고, 비동기 처리, 배칭, 로드 밸런싱 등을 구현합니다. 프로덕션 환경에서는 컨테이너화(Docker), 오케스트레이션(Kubernetes), 모니터링, 오토스케일링 등의 인프라 구성이 필요합니다.

평가와 배포

LLM 평가는 자동 메트릭과 인간 평가를 조합하여 수행합니다. Perplexity는 언어 모델이 텍스트를 얼마나 잘 예측하는지를 측정하며, 낮을수록 좋습니다. BLEU는 생성 텍스트와 참조 텍스트 간의 n-gram 일치도를 측정하며 기계 번역에서 주로 사용됩니다. ROUGE는 요약 태스크에서 사용되며, 참조 텍스트와의 n-gram 재현율을 계산합니다. BERTScore는 BERT 임베딩을 사용하여 의미적 유사도를 측정합니다.

LLM-as-a-Judge는 GPT-4 등의 강력한 모델이 다른 모델의 출력을 평가하는 방법입니다. 인간 평가와 높은 상관관계를 보이면서도 비용과 시간을 대폭 절약할 수 있습니다. 단일 점수 매기기, 쌍별 비교, 참조 기반 평가 등 다양한 방식으로 적용됩니다. 위치 편향(첫 번째 응답을 선호하는 경향), 자기 편향(자신이 생성한 텍스트를 선호하는 경향), 길이 편향(긴 답변을 선호하는 경향) 등의 한계가 있으므로, 순서를 무작위로 섞거나 여러 평가 모델을 앙상블하는 등의 보정이 필요합니다.

RAGAS는 RAG 시스템의 성능을 평가하는 프레임워크입니다. Faithfulness는 생성된 답변이 검색된 문맥에 근거하는지를 측정합니다. Answer Relevancy는 답변이 질문에 대해 얼마나 관련이 있는지를 평가합니다. Context Precision은 검색된 문맥 중 관련 있는 항목이 상위에 위치하는지를 측정합니다. Context Recall은 질문에 답하기 위해 필요한 정보가 검색 결과에 포함되어 있는지를 평가합니다. 이러한 메트릭을 종합적으로 분석하면 RAG 시스템의 어떤 부분을 개선해야 하는지 파악할 수 있습니다.

모델 배포 시에는 양자화, 배칭, KV 캐시 최적화 등을 통해 추론 효율을 높입니다. FastAPI는 비동기 지원과 자동 API 문서화로 LLM API 서버 구축에 적합합니다. Streamlit은 Python만으로 웹 인터페이스를 빠르게 구현할 수 있어, 데모와 프로토타이핑에 널리 사용됩니다. Gradio는 ML 모델의 인터랙티브한 데모를 만드는 데 특화되어 있습니다. 프로덕션 배포 시에는 입력 검증, 에러 처리, 로깅, 레이트 리미팅, 인증 등의 보안 및 운영 요소를 반드시 고려해야 합니다. A/B 테스트를 통해 모델 업데이트의 효과를 검증하고, 지속적인 모니터링으로 모델 드리프트를 감시하는 것이 장기 운영의 핵심입니다.
